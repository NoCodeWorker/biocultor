# Editorial

## Objetivo

La línea editorial de `app/(shop)/aprende/*` debe convertir evidencia externa en contenido que refuerce intención de compra real.

No se publican artículos para “rellenar SEO”.

No se publican artículos con pseudo-ciencia.

No se publican artículos con claims cerrados que el estudio no sostenga.

## Principio

Cada artículo debe responder esta secuencia:

1. Qué estudió la fuente.
2. Qué observó realmente.
3. Qué significa eso para el contexto de uso.
4. Cómo ayuda a elegir formato, aplicación o criterio de compra.

## Fuentes válidas

- universidades
- papers revisados por pares
- journals científicos o técnicos
- centros de investigación
- publicaciones técnicas institucionales

## Fuentes no válidas como base principal

- copy comercial de terceros
- blogs sin autoría o institución clara
- resúmenes sin enlace a fuente primaria
- contenido generado por IA sin referencia verificable

## Plantilla de artículo

### 1. Apertura

- problema o pregunta de compra concreta
- una frase clara sobre por qué ese tema importa al usuario

Ejemplo:

`¿Tiene sentido aplicar té de humus en cítricos con riego localizado? Esta guía parte de estudios sobre manejo y contexto de uso para ayudarte a comprar con más criterio.`

### 2. Fuente principal

Campos obligatorios:

- título del estudio o paper
- autoría o institución
- año
- enlace o referencia primaria

Formato sugerido:

`Fuente base: [Título del estudio] — [Universidad / Journal / Institución], [Año].`

### 3. Qué observó el estudio

Reglas:

- resumir solo lo que la fuente sostiene
- evitar extrapolaciones de producto
- usar lenguaje observacional

Buenas fórmulas:

- `el estudio observa`
- `los autores describen`
- `la publicación señala`
- `en este contexto se analiza`

Evitar:

- `queda demostrado que Biocultor...`
- `esto garantiza...`
- `la planta responderá...`

### 4. Qué significa para el comprador

Objetivo:

- traducir la evidencia a criterio de compra

Preguntas que debe responder:

- qué tipo de uso tiene sentido
- qué dudas reduce
- qué formato podría encajar mejor
- qué limitaciones o cautelas hay

### 5. Conexión comercial

La conexión con compra debe ser explícita pero sobria.

Debe enlazar con:

- ficha de producto
- landing por cultivo o aplicación
- contenido relacionado si ayuda a cerrar la decisión

Buenas fórmulas:

- `si tu caso encaja con este escenario de uso, revisa la ficha del producto`
- `si trabajas con este tipo de cultivo, compara formatos aquí`
- `si tu duda principal es el sistema de riego, revisa esta guía relacionada`

### 6. Citas visibles

Cada artículo debe incluir al menos:

- una cita visible a la autoridad del estudio
- una sección final de referencias

Formato mínimo:

`Referencias`

- `[Autoría o institución]. [Título]. [Año]. [Enlace o referencia].`

## Estructura editorial mínima

Cada artículo de blog debe incluir:

- `keyword` principal
- intención de búsqueda
- fuente principal identificada
- resumen del hallazgo
- implicación práctica
- enlace a compra
- referencias

## Checklist antes de publicar

- la fuente principal está identificada
- la autoridad de la fuente aparece visible
- el artículo distingue entre observación y conclusión comercial
- no hay promesas cerradas ni resultados garantizados
- no se usan porcentajes o mejoras si no vienen de la fuente y del contexto correcto
- la CTA conecta con producto, formato o landing relevante
- existe sección de referencias

```md
- Autoridad:

- Enlace:

## Qué observó la fuente

[Resumen fiel, sin exagerar]

## Qué significa para este tipo de comprador

[Traducción práctica]

## Cómo se conecta con Biocultor

[Formato, aplicación, compatibilidad o criterio de compra]

```

## Regla final

La evidencia debe reforzar la compra.

Nunca debe disfrazar la compra de evidencia.
